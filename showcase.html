Architecture

시스템 구조

Python + slack-bolt (Socket Mode) 기반의 간결한 아키텍처

slack-multi-agent/ ├── slack_bot.py # Socket Mode 이벤트 수신, 채널별 라우팅 ├── watchdog.py # 크래시 감지, 자동 재시작, 원격 제어 ├── cancel.py # 작업 취소 + 프로세스 관리 ├── agents/ │ ├── base.py # Agent 공통 (subprocess, 타임아웃) │ ├── claude.py # Claude Code (stream-json, 토큰 추적) │ ├── codex.py # Codex CLI (노이즈 필터, exec 모드) │ ├── gemini.py # Gemini CLI (YOLO 모드, quota 재시도) │ ├── claude_backup.py # Claude-B (다른 관점 답변) │ └── codex_backup.py # Codex-B (비판적 리뷰) ├── modes/ │ ├── debate.py # 토론 (합의 감지, 라운드, 교착 감지) │ ├── coding.py # 코딩 (파이프라인 + 병렬 모드) │ └── bridge.py # Bridge (폴더별 CLI 직접 연결) ├── tests/ # 53개 테스트 (합의, 교체, 통합) ├── config.py # 환경 변수, 타임아웃 설정 └── requirements.txt # slack-bolt, python-dotenv

Tech Stack

🐍 Python 3.11+
⚡ slack-bolt
🔌 Socket Mode
📦 subprocess

CLI 호출 방식

# Claude (stream-json + 토큰 추적)
type "tmp.txt" | claude -p --output-format stream-json

# Codex (full-auto + 노이즈 필터)
type "tmp.txt" | codex exec --full-auto

# Gemini (YOLO 모드 + quota 재시도)
type "tmp.txt" | gemini -y -p ""

Features

주요 기능

멀티 에이전트 시스템의 핵심 기능들

🤝

자동 합의 감지

전원 합의 시 즉시 종료, 5라운드 이상에서 다수 합의 시 조기 종료. 최대 10라운드 후 다수결로 결론 도출.

🔄

라운드별 랜덤 순서

매 라운드마다 발언 순서를 랜덤으로 셔플하여 특정 AI의 편향을 방지하고 다양한 토론 흐름을 생성합니다.

🧵

스레드 추가 질문

스레드에 추가 질문 시 원본 주제를 포함한 맥락 기반 추가 토론을 진행하며, 합의에 도달할 때까지 계속됩니다.

🛡

타임아웃 백업 에이전트

Claude 타임아웃 시 Codex-B(🟤), Codex/Gemini 타임아웃 시 Claude-B(🟡)가 다른 관점으로 자동 대체 답변.

🔍

교착 상태 감지

토론이 교착 상태에 빠지면 이를 자동으로 감지하고 새로운 관점을 주입하여 논의를 진전시킵니다.

📋

이전 합의 자동 참조

당일 이전 토론의 합의 결론을 자동으로 참조하여 맥락이 이어지는 연속적인 토론이 가능합니다.

🚨

병렬 실행 모드

메시지에 에이전트 이름을 지정하면 동시 병렬 실행. 완료 후 Codex가 최종 보고서를 작성합니다.

🛑

!stop 강제 중단

스레드에서 !stop → 해당 작업만 취소. 채널 최상위 → 전체 작업 취소. subprocess도 즉시 kill.

🚨

Watchdog 크래시 복구

봇 프로세스 감시 + 크래시 시 5초 후 자동 재시작 + Slack 알림. !bot status/restart로 원격 제어.

📊

실시간 진행 표시

15초마다 경과 시간 + 작업 미리보기를 업데이트. Claude는 토큰 사용량(입력/캐시/출력 + USD)도 표시.

Flow

워크플로우

토론, 코딩, Bridge 세 가지 모드의 상세 흐름

Step 1

사용자가 #ai-협업에 메시지 작성

토론 주제를 채널에 입력하면 봇이 스레드를 생성하고 토론 시작을 안내합니다.

Step 2

3개 AI에게 질문 전달

각 AI에게 비동기로 질문을 전달하고, 이전 답변들을 컨텍스트로 포함합니다. 매 라운드마다 순서를 랜덤 셔플합니다.

Claude Codex Gemini
Step 3

스레드에서 자유 토론 진행

각 AI가 [Claude], [Codex], [Gemini] 이름으로 스레드에 답변합니다. 이전 응답을 참고하여 반론, 동의, 보충 의견을 제시합니다.

Step 4

합의 감지 및 결론 도출

각 응답에 포함된 합의 메타데이터를 파싱하여 전원 합의(즉시 종료), 다수 합의(5라운드+), 최대 라운드 도달 시 다수결로 결론을 도출합니다.

Step 5

결론 요약 포스팅

합의된 결론을 채널(스레드 밖)에 요약 포스팅합니다. 이 결론은 당일 이후 토론에서 자동으로 참조됩니다.

Phase 1 — 기획 + 개발

Claude가 설계하고 코드를 작성

사용자의 요청을 분석하여 기획, 설계, 코드 작성을 수행하고 스레드에 결과를 포스팅합니다.

Claude → 기획 + 개발
Phase 2 — 코드 리뷰

Codex가 코드를 리뷰

Claude가 작성한 코드를 Codex가 리뷰하고, 개선점이나 버그를 지적합니다.

Codex → 코드 리뷰
Phase 3 — 테스트

전원 참여 테스트

Codex가 테스트 리더로서 테스트를 설계/작성하고, Claude와 Gemini도 테스트에 참여합니다.

Codex → 테스트 리더 Claude → 테스트 Gemini → 테스트
Phase 4 — 이슈 수정 (선택)

이슈 발견 시 반복 수정

테스트에서 이슈가 발견되면 Claude에게 수정을 요청합니다. 최대 3회까지 반복 가능합니다.

최대 3회 반복
Complete

Codex 최종 보고서

Codex가 전체 결과를 취합하여 핵심 요약 + 발견 사항 + 개선 제안을 포함한 최종 보고서를 작성합니다.

Codex → 최종 보고서
Step 1

프로젝트 채널에 메시지 작성

#ai-stockradar 또는 #ai-tc_manager에 메시지를 보내면 해당 프로젝트 폴더에서 CLI가 실행됩니다.

Step 2

Claude Code 또는 Codex 실행

기본은 claude --continue -p로 멀티턴 대화. codex: 접두어를 붙이면 Codex CLI로 전환됩니다.

기본: Claude codex: Codex
Step 3

스레드에 응답 게시

CLI 결과가 스레드에 포스팅됩니다. 4000자 초과 시 자동 분할. 스레드에서 추가 대화도 가능합니다.
